---
title: Um site estático para o trabalho
date: 2026-06-13
tags: meta, infraestrutura
summary: Por que este site são algumas centenas de linhas de Python que cospem HTML puro, e nada mais.
---

Mantenho vários projetos ao mesmo tempo — bibliotecas de criptografia, modos
digitais de HF, um toolchain para microcontroladores AVR de 8 bits. Quase tudo
que aprendo construindo eles nunca sai do histórico de commits. Este site é onde
pretendo registrar isso.

Ele é deliberadamente pequeno. Não há servidor de aplicação, nem banco de dados,
nem framework JavaScript. Um script curto em Python lê Markdown de um diretório
`content/`, renderiza com alguns templates e escreve HTML estático — em
português, inglês e espanhol — em `public/`. O GitHub Actions roda esse script a
cada push e publica o resultado no GitHub Pages.

```python
for lang in LANGS:
    posts = load_posts(lang, include_drafts)
    pages = load_pages(lang)
    # ... renderiza index, posts, páginas e um feed RSS em public/<lang>/
```

Escrever um post é adicionar um arquivo Markdown e dar push. Nada para manter no
ar, nada para corrigir, nada que possa cair às três da manhã. Para quem passa o
dia de trabalho mantendo sistemas de produção vivos, essa ausência é justamente o
ponto.

Os posts virão na medida em que os projetos vierem.
